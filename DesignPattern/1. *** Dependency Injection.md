অবশ্যই। Spring Boot এর DI (Dependency Injection) একদম সহজভাবে দেখি।

ধরো `UserService` এর database এ data save করার জন্য `DatabaseService` দরকার।

# Step 1: DatabaseService

```java
package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class DatabaseService {

    public void save() {
        System.out.println("Data Saved");
    }
}
```

---

# Step 2: UserService

```java
package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class UserService {

    private final DatabaseService databaseService;

    public UserService(DatabaseService databaseService) {
        this.databaseService = databaseService;
    }

    public void registerUser() {

        System.out.println("User Registration Started");

        databaseService.save();

        System.out.println("User Registration Completed");
    }
}
```

---

# Step 3: Controller

```java
package com.example.demo.controller;

import com.example.demo.service.UserService;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class UserController {

    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @GetMapping("/register")
    public String register() {

        userService.registerUser();

        return "User Registered";
    }
}
```

---

# Spring ভিতরে কী করে?

তুমি কোথাও লিখো নাই:

```java
new DatabaseService();
```

বা

```java
new UserService();
```

তবুও কাজ করছে।

কারণ Spring Container নিজেই object বানায়।

Spring ভিতরে এমন কিছু করে:

```java
DatabaseService databaseService =
        new DatabaseService();

UserService userService =
        new UserService(databaseService);

UserController userController =
        new UserController(userService);
```

# DI না থাকলে

```java
@Service
public class UserService {

    private DatabaseService databaseService =
            new DatabaseService();

}
```

এখানে `UserService` নিজেই object বানাচ্ছে।

```txt
UserService ----> new DatabaseService()
```

এটাকে Tight Coupling বলে।

---

# DI থাকলে

```java
public UserService(DatabaseService databaseService) {
    this.databaseService = databaseService;
}
```

এখানে object Spring inject করে।

```txt
Spring
   |
   v
DatabaseService Object
   |
   v
UserService
```

এটাই Dependency Injection।

---

## Constructor-এর এই লাইনটাই DI-এর মূল

```java
public UserService(DatabaseService databaseService) {
    this.databaseService = databaseService;
}
```

মানে:

```txt
Spring বলছে:

"এই নাও DatabaseService object,
UserService এর ভিতরে ব্যবহার করো।"
```

তখন `UserService` এর ভিতরে `DatabaseService` এর সব **public method** ব্যবহার করা যাবে:

```java
databaseService.save();
```

```java
databaseService.delete();
```

```java
databaseService.update();
```

যদি এগুলো `DatabaseService` এ থাকে। তাই DI মানে হলো **object share করা**, কিন্তু object create করার দায়িত্ব Spring Container-এর।
