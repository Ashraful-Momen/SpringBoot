# 1. model Folder

## Path

```txt
com.unisoftai.lms.model
```

---

## Student.java

```java
package com.unisoftai.lms.model;

import jakarta.persistence.*;

@Entity
@Table(name = "students")
public class Student {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private String id;

    @Column(nullable = false)
    private String name;

    @Column(unique = true)
    private String email;

    private String department;

    public Student() {}

    public Student(String name, String email, String department) {
        this.name = name;
        this.email = email;
        this.department = department;
    }

    // Getter Setter

    public String getId() { return id; }
    public void setId(String id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    public String getDepartment() { return department; }
    public void setDepartment(String department) { this.department = department; }
}
```

### Key Points

- `@Id` + `@GeneratedValue(strategy = GenerationType.UUID)` — Auto-generate UUID as primary key (String type)
- `@Column(nullable = false)` — name field mandatory
- `@Column(unique = true)` — email must be unique
- `@Table(name = "students")` — maps to `students` table in PostgreSQL
- No-arg constructor required by JPA/Hibernate

## Path

```txt
com.unisoftai.lms.repository
```

## StudentRepository.java

```java
package com.unisoftai.lms.repository;

import com.unisoftai.lms.model.Student;
import org.springframework.data.jpa.repository.JpaRepository;

public interface StudentRepository extends JpaRepository<Student, String> {
}
```

### Key Points

- `interface` — not a class! Spring Data JPA auto-generates the implementation
- `JpaRepository<Student, String>` — `String` is the ID type (UUID stored as String)
- এই empty interface-ই sufficient — `findAll()`, `save()`, `findById()`, `delete()` সব automatically পাওয়া যায়

# 3. service Folder

এখানে business logic থাকে।

---

## StudentService.java

```java
package com.unisoftai.lms.service;

import com.unisoftai.lms.model.Student;
import com.unisoftai.lms.repository.StudentRepository;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class StudentService {

    private final StudentRepository studentRepository;

    public StudentService(StudentRepository studentRepository) {
        this.studentRepository = studentRepository;
    }

    // Create Student
    public Student createStudent(Student student) {
        return studentRepository.save(student);
    }

    // Get All Students
    public List<Student> getAllStudents() {
        return studentRepository.findAll();
    }
}
```

### Key Points

- `@Service` — Spring component, business logic layer
- **Constructor Injection** (not `@Autowired`) — modern best practice, Spring auto-injects the dependency
- `studentRepository` is `final` — ensures immutability

# 4. controller Folder

---

## StudentController.java

```java
package com.unisoftai.lms.controller;

import com.unisoftai.lms.model.Student;
import com.unisoftai.lms.service.StudentService;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/students")
public class StudentController {

    private final StudentService studentService;

    public StudentController(StudentService studentService) {
        this.studentService = studentService;
    }

    // Create Student
    @PostMapping
    public Student createStudent(@RequestBody Student student) {
        return studentService.createStudent(student);
    }

    // Get All Students
    @GetMapping
    public List<Student> getAllStudents() {
        return studentService.getAllStudents();
    }
}
```

### Key Points

- `@RestController` = `@Controller` + `@ResponseBody` (auto JSON conversion)
- `@RequestMapping("/api/students")` — base path `/api/students`
- `@PostMapping` — handles POST requests
- `@GetMapping` — handles GET requests
- `@RequestBody` — JSON body → Java object (Jackson auto-converts)
- **Constructor Injection** — same best practice as service layer

# Database Configuration

## application.properties

```properties
spring.application.name=lms

# PostgreSQL (Docker)
spring.datasource.url=jdbc:postgresql://localhost:5433/lms_db
spring.datasource.username=lms_user
spring.datasource.password=lms_password
spring.datasource.driver-class-name=org.postgresql.Driver

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true

# Redis (Docker)
spring.data.redis.host=localhost
spring.data.redis.port=6380
```

### Key Points

- PostgreSQL 16 running on Docker (host port `5433` → container port `5432`)
- `ddl-auto=update` — Hibernate automatically creates/updates tables from entity classes
- `show-sql=true` — SQL queries printed in console (dev only)
- Redis configured for future caching/sessions

# Beginner Mistake vs Correct Practice

### Bad Practice — সব code controller এ লিখে

```txt
Controller → Database Directly
```

### Correct Practice — Layer separation

```txt
Controller → Service → Repository → Database
```

# Dependency Injection: @Autowired vs Constructor

### Old Way (field injection) — avoid this

```java
@Autowired
private StudentRepository studentRepository;
```

### Best Practice (constructor injection) — use this

```java
private final StudentRepository studentRepository;

public StudentService(StudentRepository studentRepository) {
    this.studentRepository = studentRepository;
}
```

**Why?** Immutable (`final`), easier to test, Spring recommended approach since Spring 4.3+.

---

# ID Strategy: UUID vs Auto-Increment

### Auto-Increment (Long)

```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

### UUID (String) — used in this project

```java
@Id
@GeneratedValue(strategy = GenerationType.UUID)
private String id;
```

**Why UUID?** Better for distributed systems, no sequential guessing, safe for public APIs. LMS project uses UUID across all 37 tables.
