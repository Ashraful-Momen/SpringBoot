# Spring Boot DI & IoC Container — এমনভাবে বুঝো যাতে আর কখনও ভুল না হয়

---

# 1. Application Start

ধরো তোমার Main Class:

```java
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {

        SpringApplication.run(
                DemoApplication.class,
                args
        );
    }
}
```

যখন চালাও:

```bash
java -jar app.jar
```

তখন এই লাইন execute হয়:

```java
SpringApplication.run(
        DemoApplication.class,
        args
);
```

এখান থেকেই Spring Boot Engine চালু হয়।

# 3. Component Scan শুরু হয়

Spring দেখে:

```java
@SpringBootApplication
```

এর ভিতরে আছে:

```java
@ComponentScan
```

Spring Main Class এর package থেকে scan শুরু করে।

```txt
com.example.demo
    |
    +--- controller
    |
    +--- service
    |
    +--- repository
```

---

ধরো:

```java
package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class DatabaseService {

}
```

Spring Scan করে।

```txt
Scanning...

DatabaseService
   |
   +-- @Service Found
```

---

আরেকটা:

```java
@Service
public class UserService {

}
```

Spring আবার খুঁজে পেল।

```txt
Scanning...

UserService
   |
   +-- @Service Found
```

# 5. Bean Map তৈরি হয়

Spring একটা Registry তৈরি করে।

ভিতরে conceptually এমন:

```java

```

```java
}
```

# 7. Constructor Analyze করে

এখন Spring দেখে:

```java

    private final DatabaseService databaseService;

    public UserService(
            DatabaseService databaseService
    ) {
        this.databaseService =
                databaseService;
    }
}
```


# Spring ভিতরে বাস্তবে প্রায় এমন করে

```java
DatabaseService db =
        new DatabaseService();

UserService userService =
        new UserService(db);

UserController controller =
        new UserController(userService);
```

কিন্তু তুমি লিখো:

```java
@Service
class DatabaseService {}

@Service
class UserService {}

@RestController
class UserController {}
```

বাকি সব Spring করে।

# Interview-এর জন্য Golden Summary

```txt
1. Application Start

2. ApplicationContext Create

3. Component Scan

4. @Service/@Repository/@Controller Detect

5. BeanDefinition Create

6. Bean Registry Build

7. Bean Object Create

8. Container এ Store

9. Constructor Dependency Analyse

10. Required Bean Search

11. Dependency Inject

12. Final Bean Store

13. Request এ Bean Reuse
```

এক লাইনে:

```txt
Spring Container (ApplicationContext)
        |
        +--> Scan Classes
        |
        +--> Create Beans
        |
        +--> Store Beans
        |
        +--> Inject Dependencies
        |
        +--> Provide Beans When Needed
```

এটাই Spring-এর **IoC (Inversion of Control)** এবং **DI (Dependency Injection)** এর সম্পূর্ণ জীবনচক্র। যদি এটা মাথায় রাখতে পারো, তাহলে Spring Boot-এর ৭০-৮০% magic আর magic মনে হবে না; Spring আসলে Bean scan করে, map/registry বানায়, object create করে, container-এ store করে, তারপর constructor-এর মাধ্যমে dependency inject করে।
