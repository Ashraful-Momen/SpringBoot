Spring Boot-এ Application Start হওয়ার সময় **Spring IoC Container** (Application Context) object তৈরি করে এবং manage করে।

সহজভাবে Flow টা:

```txt
Application Start
        |
        v
Spring Container (ApplicationContext)
        |
        v
@Component Scan
        |
        v
@Service
@Repository
@Controller
@Component
খুঁজে বের করে
        |
        v
Object (Bean) Create করে
        |
        v
Container এর ভিতরে Store করে
```

---

## কে Object Create করে?

Spring Container।

উদাহরণ:

```java
@Service
public class DatabaseService {

}
```

তুমি কোথাও লিখো নাই:

```java
new DatabaseService();
```

কিন্তু Spring Startup এর সময় নিজেই করে:

```java
DatabaseService databaseService =
        new DatabaseService();
```

এবং Container এ রাখে।

---

## কে Dependency Provide করে?

এটাও Spring Container।

ধরো:

```java
@Service
public class UserService {

    private final DatabaseService databaseService;

    public UserService(DatabaseService databaseService) {
        this.databaseService = databaseService;
    }
}
```

Spring দেখে:

```txt
UserService এর জন্য DatabaseService দরকার
```

তখন Container থেকে `DatabaseService` Bean বের করে Constructor এ Inject করে।

ভেতরে ভেতরে এমন কিছু হয়:

```java
DatabaseService db = container.getBean(DatabaseService.class);

UserService userService =
        new UserService(db);
```

---

## Container আসলে কী?

Spring Boot এ Container সাধারণত:

ApplicationContext

এবং এর implementation যেমন:

* AnnotationConfigApplicationContext
* WebApplicationContext

এসব।

## Real Flow

```java
@Service
public class DatabaseService {
}
```

```java
@Service
public class UserService {

    private final DatabaseService databaseService;

    public UserService(DatabaseService databaseService) {
        this.databaseService = databaseService;
    }
}
```

Startup এর সময় Spring প্রায় এভাবে চিন্তা করে:

```txt
1. DatabaseService পেলাম
2. Object বানালাম
3. Container এ রাখলাম

4. UserService পেলাম
5. Constructor এ DatabaseService লাগবে
6. Container থেকে DatabaseService বের করলাম
7. UserService Object বানালাম
8. Container এ রাখলাম
```

**Question:** Spring এ object create করে কে?

**Answer:**

```txt
Spring IoC Container (ApplicationContext)
```

---

**Question:** Dependency inject করে কে?

**Answer:**

```txt
Spring IoC Container (ApplicationContext)
```

---

**Question:** Bean কোথায় store থাকে?

**Answer:**

```txt
Spring IoC Container / ApplicationContext এর ভিতরে
```

এক লাইনে:

> **Spring Container Bean create করে, manage করে, store করে এবং প্রয়োজন হলে dependency inject করে দেয়।**
