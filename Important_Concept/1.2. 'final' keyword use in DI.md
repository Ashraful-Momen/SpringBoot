খুব ভালো প্রশ্ন।

এই কোডে:

```java
private final DatabaseService databaseService;
```

`final` এর মানে হলো:

> একবার value assign করা হলে পরে আর change করা যাবে না।

---

## Java Example

```java
final int age = 25;

age = 30; // Error
```

কারণ `age` final।

---

## Object Reference এর ক্ষেত্রে

```java
DatabaseService db1 = new DatabaseService();
DatabaseService db2 = new DatabaseService();

final DatabaseService databaseService = db1;
```

এখন:

```java
databaseService = db2; // Error
```

কারণ reference change করা যাবে না।

---

## Spring DI Example

```java
@Service
public class UserService {

    private final DatabaseService databaseService;

    public UserService(DatabaseService databaseService) {
        this.databaseService = databaseService;
    }
}
```

Spring যখন object inject করে:

```java
DatabaseService db = new DatabaseService();

UserService userService =
        new UserService(db);
```

তখন:

```java
this.databaseService = db;
```

একবার assign হয়ে গেল।

এরপর আর:

```java
this.databaseService = new DatabaseService();
```

করতে পারবে না।

---

## তাহলে Method Call করা যাবে?

অবশ্যই।

```java
databaseService.save();
databaseService.delete();
databaseService.update();
```

এসব করা যাবে।

`final` object কে lock করে না।

`final` শুধু reference change হওয়া বন্ধ করে।

---

উদাহরণ:

```java
final List<String> names = new ArrayList<>();

names.add("Ashraful");
names.add("Rahim");
```

এটা ঠিক আছে।

কিন্তু:

```java
names = new ArrayList<>();
```

এটা Error।

---

## Spring Boot এ `final` কেন ব্যবহার করা হয়?

কারণ dependency inject হওয়ার পরে সেটি আর পরিবর্তন হওয়া উচিত না।

```java
private final UserRepository userRepository;
private final EmailService emailService;
private final RedisService redisService;
```

এগুলো application চলাকালীন একই dependency ব্যবহার করবে।

এতে code:

* Safe হয়
* Immutable হয়
* Accidentally dependency replace হওয়ার risk থাকে না

---

সহজভাবে মনে রাখো:

```java
private final DatabaseService databaseService;
```

মানে:

```txt
DatabaseService object ব্যবহার করা যাবে ✔

DatabaseService reference পরিবর্তন করা যাবে না ❌
```

Spring Boot-এ Constructor Injection এর সাথে `final` ব্যবহার করা বর্তমানে best practice।
