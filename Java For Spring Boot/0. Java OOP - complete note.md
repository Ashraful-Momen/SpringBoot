# Object-Oriented Programming (OOP) in Java 21 for Spring Boot ☕🚀

## What are the four pillars of OOP?

**Answer:**
Encapsulation, Abstraction, Inheritance, and Polymorphism.

These four concepts are heavily used in Spring Boot applications for building clean, reusable, and maintainable backend systems.

---

# What is a Class?

## Answer:

A class is a blueprint used to create objects.
It defines:

* Fields (variables/data)
* Methods (behaviors/functions)

In Java 21, classes are used everywhere in Spring Boot:

* Entity classes
* Service classes
* Controller classes
* DTO classes

### Example:

```java
public class Student {

    String name;
    int id;
    String email;

    void showInfo() {
        System.out.println("Name: " + name);
        System.out.println("ID: " + id);
        System.out.println("Email: " + email);
    }
}
```

### Usage:

```java
public class Main {
    public static void main(String[] args) {

        Student s1 = new Student();

        s1.name = "John";
        s1.id = 101;
        s1.email = "john@example.com";

        s1.showInfo();
    }
}
```

### Output:

```text
Name: John
ID: 101
Email: john@example.com
```

# What is an Object?

## Answer:

An object is an instance of a class.

A class is only a blueprint.
The object holds real data.

### Example:

```java
Student s1 = new Student();
```

Here:

* `Student` = class
* `s1` = object

### Full Example:

```java
public class Student {

    String name;
    int id;

    void showInfo() {
        System.out.println(name + " - " + id);
    }

    public static void main(String[] args) {

        Student s1 = new Student();

        s1.name = "Alice";
        s1.id = 102;

        s1.showInfo();
    }
}
```

### Output:

```text
Alice - 102
```

# What is a Constructor?

## Answer:

A constructor is a special method used to initialize objects.

Rules:

* Same name as class
* No return type
* Automatically called during object creation

---

# Types of Constructor

## 1. Default Constructor

```java
public class Student {

    String name;
    int id;

    Student() {
        name = "Unknown";
        id = 0;
    }

    void showInfo() {
        System.out.println(name + " - " + id);
    }

    public static void main(String[] args) {

        Student s1 = new Student();
        s1.showInfo();
    }
}
```

### Output:

```text
Unknown - 0
```

---

## 2. Parameterized Constructor

```java
public class Student {

    String name;
    int id;

    Student(String name, int id) {
        this.name = name;
        this.id = id;
    }

    void showInfo() {
        System.out.println(name + " - " + id);
    }

    public static void main(String[] args) {

        Student s1 = new Student("Karim", 104);
        s1.showInfo();
    }
}
```

### Output:

```text
Karim - 104
```

# What is an Access Modifier?

## Answer:

Access modifiers control visibility and accessibility.

| Modifier  | Access Level                               |
| --------- | ------------------------------------------ |
| public    | Accessible everywhere                      |
| private   | Accessible only inside same class          |
| protected | Accessible in same package + child classes |
| default   | Same package only                          |

---

## Example:

```java
public class Student {

    public String name;
    private int id;
    protected String email;

    public void setId(int id) {
        this.id = id;
    }

    public void showInfo() {
        System.out.println(name + " " + id + " " + email);
    }
}
```

### Usage:

```java
Student s1 = new Student();

s1.name = "Tanvir";

// s1.id = 10; ❌ Error

s1.setId(10);

s1.showInfo();
```

# What is Encapsulation?

## Answer:

Encapsulation means hiding data and controlling access using methods.

Usually:

* fields → private
* methods → public getter/setter

This is one of the MOST IMPORTANT concepts in Spring Boot.

---

## Example:

```java
public class Student {

    private String name;
    private int id;

    public void setName(String name) {
        this.name = name;
    }

    public void setId(int id) {
        this.id = id;
    }

    public void showInfo() {
        System.out.println(name + " - " + id);
    }

    public static void main(String[] args) {

        Student s1 = new Student();

        s1.setName("Rahim");
        s1.setId(103);

        s1.showInfo();
    }
}
```

### Output:

```text
Rahim - 103
```

# What are Getter and Setter?

## Answer:

Getter and Setter methods provide controlled access to private fields.

* Setter → sets value
* Getter → returns value

---

## Example:

```java
public class Student {

    private String name;
    private int id;

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public void setId(int id) {
        this.id = id;
    }

    public int getId() {
        return id;
    }

    public static void main(String[] args) {

        Student s1 = new Student();

        s1.setName("Amin");
        s1.setId(106);

        System.out.println(s1.getName());
        System.out.println(s1.getId());
    }
}
```

### Output:

```text
Amin
106
```

# What is Inheritance?

## Answer:

Inheritance allows a child class to inherit fields and methods from a parent class.

Uses:

* Code reuse
* Clean architecture
* Extensibility

---

## Example:

```java
class Course {

    String title;
    int duration;

    void enrollCourse() {
        System.out.println("Enrolled in " + title);
    }
}

class JobInterviewCourse extends Course {

    void prepareInterview() {
        System.out.println("Preparing for interview");
    }
}

public class Main {

    public static void main(String[] args) {

        JobInterviewCourse course = new JobInterviewCourse();

        course.title = "Java Interview";
        course.duration = 40;

        course.enrollCourse();
        course.prepareInterview();
    }
}
```

### Output:

```text
Enrolled in Java Interview
Preparing for interview
```

# What is Polymorphism?

## Answer:

Polymorphism means “many forms”.

Two Types:

1. Method Overloading
2. Method Overriding

---

# Method Overloading (Compile-time Polymorphism)

## Answer:

Same method name but different parameters.

---

## Example:

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
}
```

### Usage:

```java
Calculator c = new Calculator();

System.out.println(c.add(5, 10));
System.out.println(c.add(5.5, 3.2));
```

### Output:

```text
15
8.7
```

---

# Why is Overloading Called Compile-time Polymorphism?

## Answer:

Because the compiler decides which method to call during compilation.

This is called:

* Static Binding
* Early Binding

---

# What is Static Binding?

## Answer:

Static binding means method calls are resolved during compile time.

Used with:

* Method overloading
* Static methods
* Final methods
* Private methods

---

# Method Overriding (Runtime Polymorphism)

## Answer:

A child class changes the behavior of a parent class method.

---

## Example:

```java
class Course {

    void showDetails() {
        System.out.println("General Course");
    }
}

class JavaCourse extends Course {

    @Override
    void showDetails() {
        System.out.println("Java Spring Boot Course");
    }
}

public class Main {

    public static void main(String[] args) {

        Course c = new JavaCourse();

        c.showDetails();
    }
}
```

### Output:

```text
Java Spring Boot Course
```

---

# Why is Overriding Called Runtime Polymorphism?

## Answer:

Because JVM decides at runtime which method to execute based on actual object type.

This is called:

* Dynamic Binding
* Late Binding

---

# What is Dynamic Binding?

## Answer:

Dynamic binding means method calls are resolved during runtime.

Used mainly in:

* Method overriding
* Runtime polymorphism

---

# Difference Between Overloading and Overriding

| Feature           | Overloading  | Overriding     |
| ----------------- | ------------ | -------------- |
| Method Name       | Same         | Same           |
| Parameters        | Different    | Same           |
| Class             | Same class   | Parent + Child |
| Decision Time     | Compile time | Runtime        |
| Polymorphism Type | Compile-time | Runtime        |

# What is a Virtual Method in Java?

## Answer:

Java does not use the `virtual` keyword like C#.

In Java:

* Every non-static, non-final, non-private method is virtual by default.

Example:

```java
class Course {

    void showDetails() {
        System.out.println("General");
    }
}

class JavaCourse extends Course {

    @Override
    void showDetails() {
        System.out.println("Java Course");
    }
}
```

Abstraction means hiding implementation details and showing only important features.

Achieved using:

* Abstract class

# What is an Abstract Class?

## Answer:

An abstract class cannot be instantiated directly.

It may contain:

* Abstract methods
* Normal methods

---

## Example:

```java
abstract class Course {

    abstract void showDetails();
}

class CompetitiveProgrammingCourse extends Course {

    @Override
    void showDetails() {
        System.out.println("Competitive Programming Course");
    }
}

public class Main {

    public static void main(String[] args) {

        Course c = new CompetitiveProgrammingCourse();

        c.showDetails();
    }
}
```

### Output:

```text
Competitive Programming Course
```

# What is an Interface?

## Answer:

An interface is a contract.

It defines what a class must do.

Java 21 heavily uses interfaces in:

* Spring Boot
* Dependency Injection
* Microservices

---

## Example:

```java
interface PaymentService {

    void pay();
}

class BkashPayment implements PaymentService {

    @Override
    public void pay() {
        System.out.println("Paid using bKash");
    }
}

public class Main {

    public static void main(String[] args) {

        PaymentService p = new BkashPayment();

        p.pay();
    }
}
```

### Output:

```text
Paid using bKash
```

# What is `this` Keyword?

## Answer:

`this` refers to current object.

Used for:

* Access current object's variables
* Constructor chaining
* Avoid naming conflicts

---

## Example:

```java
class Student {

    String name;

    Student(String name) {
        this.name = name;
    }

    void show() {
        System.out.println(this.name);
    }
}
```

---

# What is `super` Keyword?

## Answer:

`super` refers to parent class object.

Used for:

* Calling parent constructor
* Accessing parent methods/fields

---

## Example:

```java
class Course {

    Course() {
        System.out.println("Course Constructor");
    }
}

class JavaCourse extends Course {

    JavaCourse() {
        super();
        System.out.println("Java Constructor");
    }
}
```

### Output:

```text
Course Constructor
Java Constructor
```

# What is a Record in Java 21?

## Answer:

Record is a modern Java feature used for immutable data classes.

Very useful in Spring Boot DTOs and API responses.

---

## Example:

```java
public record StudentDTO(
        String name,
        int id,
        String email
) {}
```

### Usage:

```java
StudentDTO s =
        new StudentDTO("Rahim", 1, "r@test.com");

System.out.println(s.name());
```

---

# Why Records are Important in Spring Boot?

## Answer:

Records reduce boilerplate code.

Automatically generates:

* constructor
* getter
* toString()
* equals()
* hashCode()

Perfect for:

* DTO
* Request Body
* Response Body

# What is Dependency Injection (Very Important for Spring Boot)?

## Answer:

Dependency Injection (DI) means Spring automatically provides required objects.

Instead of:

```java
Service s = new Service();
```

Spring creates/manages objects automatically.

This is called:

* IoC (Inversion of Control)
* Dependency Injection

---

# Example of Dependency Injection in Spring Boot

```java
@Service
class CourseService {

    void show() {
        System.out.println("Course Service");
    }
}

@RestController
class CourseController {

    private final CourseService courseService;

    CourseController(CourseService courseService) {
        this.courseService = courseService;
    }

    @GetMapping("/")
    public String home() {

        courseService.show();

        return "Hello";
    }
}
```
