# ১. @Configuration কি?

ধরো তোমার কাছে একটা class আছে:

```java
public class SmsService {

    public void send(String message) {
        System.out.println(message);
    }
}
```

এখন Spring কে বলতে হবে:

```text
এই SmsService object টা startup এর সময় create করে container এ রাখো।
```

কিন্তু SmsService class এ তুমি

```java
@Component
```

দাওনি।

তাহলে Spring জানবে কিভাবে object create করতে হবে?

---

তখন আমরা একটা Factory Class বানাই।

```java
@Configuration
public class AppConfig {

}
```

এখানে

```java
@Configuration
```

মানে:

```text
Spring, এই class টা special।

এই class এর ভিতরে object তৈরির নিয়ম আছে।
```

---

Laravel Equivalent

```php
AppServiceProvider
```

এর মতো।

```php
public function register()
{
   // service register করি
}
```

# ২. @Bean কি?

ধরো:

```java
@Configuration
public class AppConfig {

    @Bean
    public SmsService smsService() {

        return new SmsService();
    }
}
```

এখন Startup এর সময় Spring এটা দেখে।

---

Spring internally এমন কিছু করে:

```java
AppConfig config = new AppConfig();

SmsService sms =
      config.smsService();
```

মানে:

```java
new SmsService();
```

---

তারপর Container এ রেখে দেয়।

```text
Container

├── smsService
```

---

ASCII

```text
@Configuration

      │

      ▼

AppConfig

      │

      ▼

@Bean

      │

      ▼

new SmsService()

      │

      ▼

Container
```

# ৩. ModelMapper আসলে কি?

অনেকেই ভাবছে এটা Spring এর অংশ।

না।

এটা একটা Library।

---

ধরো Database Entity:

```java
public class User {

    private Long id;

    private String name;

    private String password;
}
```

---

Frontend এ পাঠাবে:

```java
public class UserResponse {

    private Long id;

    private String name;
}
```

---

কারণ password পাঠানো যাবে না।

---

Without ModelMapper

তোমাকে লিখতে হবে:

```java
UserResponse response =
     new UserResponse();

response.setId(
      user.getId()
);

response.setName(
      user.getName()
);
```

---

১০০ field হলে?

```text

```

---

তখন ModelMapper।

```java

      new ModelMapper();

      mapper.map(
```

Automatically:


# Startup এ ModelMapper Container এ কিভাবে গেল?

```java
@Configuration
public class AppConfig {

    @Bean
    public ModelMapper modelMapper() {

        return new ModelMapper();
    }
}
```

---

Startup

```text
main()

  │

  ▼

@Configuration Scan

  │

  ▼

Found AppConfig

  │

  ▼

Found @Bean

  │

  ▼

Call method

  │

  ▼

new ModelMapper()

  │

  ▼

Store Container
```

---

Container

```text
ApplicationContext

├── userService
├── userRepository
├── modelMapper
```

---

এরপর

```java
@Service
@RequiredArgsConstructor
public class UserService {

    private final ModelMapper modelMapper;
}
```

Spring বলে:

```text
UserService বানাতে ModelMapper লাগবে
```

Container এ খুঁজে:

```text
Found ✔
```

---

তারপর:

```java
new UserService(
      modelMapper
);
```

---

সবচেয়ে ছোট সংজ্ঞা:

### @Configuration

```text
Bean তৈরির Factory Class
```

### @Bean

```text
এই method call করে object তৈরি করো
এবং container এ রাখো
```

### ModelMapper

```text
এক object থেকে অন্য object এ data copy করার library
```

### Flow

```text
@Configuration
      │
      ▼
@Bean
      │
      ▼
new ModelMapper()
      │
      ▼
ApplicationContext
      │
      ▼
Inject into UserService
```

এখনও যদি `@Configuration` আর `@Bean` পুরোপুরি পরিষ্কার না হয়, তাহলে আমি একটা **"Spring Container manually Java code দিয়ে build করে"** দেখাতে পারি, যেখানে Spring ছাড়া plain Java দিয়ে Container, Bean, DI বানিয়ে দেখাবো। তখন ৯০% Spring Core architecture একবারেই clear হয়ে যাবে।
