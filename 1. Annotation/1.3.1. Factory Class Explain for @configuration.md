Spring Boot-এ `@Configuration` class-কে অনেক সময় **Factory Class** বলা হয়, কারণ এই class-এর কাজ হলো object (Bean) তৈরি করা এবং Spring Container-কে দেওয়া।

---

## Factory বলতে কী বুঝায়?

Factory Pattern-এর মূল idea:

```txt
Object Create করার দায়িত্ব
↓
একটা আলাদা Class-এর কাছে থাকবে
```

Normal Java:

```java
UserService userService = new UserService();
```

এখানে object create করছে caller নিজে।

কিন্তু Factory ব্যবহার করলে:

```java
UserService userService = Factory.getUserService();
```

Object create করার দায়িত্ব Factory-এর।

---

## Spring এ Factory Class

```java
@Configuration
public class AppConfig {

    @Bean
    public UserService userService() {
        return new UserService();
    }
}
```

এখানে:

```java
@Bean
public UserService userService() {
    return new UserService();
}
```

এই method-টা একটা factory method।

কারণ:

```txt
Method call
    ↓
Object create
    ↓
Spring Container এ store
```

---

## Internally কী হয়?

Application Start

```txt
@SpringBootApplication
        ↓
Component Scan
        ↓
@Configuration পাওয়া গেল
        ↓
AppConfig load হলো
        ↓
@Bean method detect হলো
        ↓
userService() call হলো
        ↓
new UserService()
        ↓
Spring Container এ store
```

---

## Example

```java
@Configuration
public class AppConfig {

    @Bean
    public UserRepository userRepository() {
        return new UserRepository();
    }

    @Bean
    public UserService userService() {
        return new UserService(userRepository());
    }
}
```

Equivalent to:

```java
UserRepository repo = new UserRepository();
UserService service = new UserService(repo);
```

কিন্তু Spring নিজে করছে।

## কেন Factory বলা হয়?

কারণ এই class:

```txt
@Configuration
```

এর ভিতরের

```txt
@Bean
```

method গুলো

```txt
Object তৈরি করে
```

এবং

```txt
Spring Container-কে দেয়
```

অর্থাৎ:

```txt
@Configuration
        ↓
Bean Factory
        ↓
Creates Objects
        ↓
Stores in IoC Container
```

## Real Internal View

ধরো:

```java
@Configuration
public class AppConfig {

    @Bean
    public UserService userService() {
        return new UserService();
    }
}
```

Spring প্রায় এমনভাবে চিন্তা করে:

```java
public class AppConfigFactory {

    public Object createBean(String beanName) {

        if(beanName.equals("userService")) {
            return new UserService();
        }

        return null;
    }
}
```

অবশ্য বাস্তবে Spring-এর ভিতরে এর জন্য আরও জটিল infrastructure আছে, যেমন:

* `BeanDefinition`
* `BeanFactory`
* `DefaultListableBeanFactory`
* `ApplicationContext`

কিন্তু conceptually:

```txt
@Configuration
        ↓
Factory Class

@Bean
        ↓
Factory Method

Created Object
        ↓
Bean
        ↓
IoC Container
```

এই কারণেই `@Configuration` class-কে Spring-এর **Java-based Bean Factory Configuration Class** বলা হয়।
