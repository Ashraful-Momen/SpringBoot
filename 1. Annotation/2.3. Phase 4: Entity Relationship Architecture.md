# Entity

## User

```java
@Entity
public class User {

    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToOne
    @JoinColumn(name = "profile_id")
    private Profile profile;

}
```

# Department Entity

```java

    @Id

}
```

# Student Entity

```java
@Entity
public class Student {

    @Id
    @GeneratedValue
    private Long id;

    @ManyToMany
    @JoinTable(
        name = "student_course",

        joinColumns =
        @JoinColumn(
            name="student_id"
        ),

        inverseJoinColumns =
        @JoinColumn(
            name="course_id"
        )
    )
    private List<Course> courses;

}
```

---

# Course Entity

# Problem

ধরো:

```java
Department department =
repository.findById(1L);
```

Department এর সাথে:

```txt
5000 Employees
```

আছে।

Question:

```txt
সব Employee কি এখনই Load হবে?
```

---

# FetchType

এটা Decide করে।

---

# Two Types

```java
FetchType.LAZY

FetchType.EAGER
```

---

# FetchType.LAZY

Need হলে Load করবে।

---

```java
@OneToMany(
   fetch = FetchType.LAZY
)
private List<Employee> employees;
```

---

# Flow

```txt
Department Load
       ↓
Employees Load হয়নি
```

---

# Access করলে

```java
department.getEmployees();
```

তখন Query Run হবে।

---

# LAZY Loading Architecture

```txt
Department
      ↓
Proxy Object
      ↓
Real Query Later
```

---

# Benefit

```txt
Less Memory
Less SQL
Better Performance
```

---

# FetchType.EAGER

Immediately Load করবে।

---

```java
@OneToMany(
    fetch = FetchType.EAGER
)
private List<Employee> employees;
```

---

# Flow

```txt
Department
      ↓
Employees
      ↓
Everything Load
```

# Problem With EAGER


# Industry Rule


# N+1 Problem

Hibernate Interview Favourite Topic।

```java
 Department d : departments
```

# Solution

```java

SELECT d
FROM Department d
JOIN FETCH d.employees
""")
```

# Cascade

Parent Save করলে Child Automatically Save হবে।

---

# Without Cascade

```java
departmentRepository.save(
    department
);

employeeRepository.save(
    employee
);
```

2 Times Save।

---

# With Cascade

```java
@OneToMany(
 cascade =
 CascadeType.ALL
)
```

---

```java
departmentRepository.save(
    department
);
```

Employee Automatically Save।

# Orphan Removal

Extremely Important।

---

# Problem

Department থেকে Employee Remove করলে Database এ কি Delete হবে?

---

Without:

```java
department.getEmployees()
.remove(employee);
```

Employee Database এ থেকে যেতে পারে।

---

# Solution

```java
@OneToMany(
   orphanRemoval = true
)
```

---

# Example

```java
@OneToMany(
    mappedBy="department",
    orphanRemoval=true
)
private List<Employee>
employees;
```

---

# Flow

```txt
Department
      ↓
Employee Removed
      ↓
Database DELETE
```

Automatically।

---

# Difference

## Cascade Remove

```txt
Delete Parent
      ↓
Delete Children
```

---

## Orphan Removal

```txt
Remove Child
      ↓
Delete Child
```

# Laravel vs Spring Boot

| Laravel Eloquent   | Hibernate          |
| ------------------ | ------------------ |
| hasOne             | @OneToOne          |
| hasMany            | @OneToMany         |
| belongsTo          | @ManyToOne         |
| belongsToMany      | @ManyToMany        |
| with()             | JOIN FETCH         |
| eager loading      | FetchType.EAGER    |
| lazy loading       | FetchType.LAZY     |
| cascade delete     | CascadeType.REMOVE |
| model event delete | orphanRemoval      |

---

# Important Annotations

| Annotation      | Purpose               |
| --------------- | --------------------- |
| @OneToOne       | One ↔ One             |
| @OneToMany      | One → Many            |
| @ManyToOne      | Many → One            |
| @ManyToMany     | Many ↔ Many           |
| @JoinColumn     | Foreign Key           |
| @JoinTable      | Bridge Table          |
| FetchType.LAZY  | Load On Demand        |
| FetchType.EAGER | Immediate Load        |
| CascadeType     | Parent → Child Action |
| orphanRemoval   | Auto Child Delete     |
