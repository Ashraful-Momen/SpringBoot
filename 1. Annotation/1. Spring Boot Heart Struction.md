# @Bean

সবচেয়ে Important।

ধরো Third Party Class:

```java
public class JwtUtil {
}
```

তুমি edit করতে পারবে না।

তাহলে:

```java
@Component
```

দিতে পারবে না।

Solution:

```java
@Configuration
public class AppConfig {

    @Bean
    public JwtUtil jwtUtil() {

        return new JwtUtil();
    }
}
```

Flow:

```text
@Configuration

 ↓

@Bean Method

 ↓

Method Execute

 ↓

Object Create

 ↓

Container Store
```

# Constructor Injection

Best Practice

```java

public class UserService {

    private final UserRepository repo;

}
```

# @RequiredArgsConstructor

Lombok Constructor Generate করে।

Without Lombok:

```java
public UserService(
        UserRepository repo) {

    this.repo = repo;
}
```

With Lombok:

```java
@RequiredArgsConstructor
@Service
public class UserService {

    private final UserRepository repo;
}
```

Generated:

```java
public UserService(
      UserRepository repo){

   this.repo = repo;
}
```

# Spring Boot Heart (এক লাইনে)

```text
@SpringBootApplication
        ↓
Component Scan
        ↓
IoC Container
        ↓
Bean Creation
        ↓
Dependency Injection
        ↓
Application Ready
```

এই Flow, IoC Container, Bean Lifecycle, Component Scan, `@Configuration + @Bean`, এবং Constructor Injection (`@RequiredArgsConstructor`) যদি পুরোপুরি বুঝে ফেলো, তাহলে Spring Boot Core Architecture-এর প্রায় 70% ভিতরের মেকানিজম তোমার পরিষ্কার হয়ে যাবে। এরপর JPA, Transaction, Security, Event, Cache—সবকিছু এই Container-এর উপরেই দাঁড়িয়ে আছে।
