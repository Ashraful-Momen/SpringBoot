# @Value কী Spring Boot এ?

`@Value` হলো Spring এর একটা annotation যেটা configuration file (`application.properties` বা `application.yml`) থেকে value এনে Java class এ inject করে।

সহজ ভাষায়:

> Code এর ভিতরে hardcode না করে config file থেকে value নিয়ে আসে।

---

# Real Life Example

ধরো তোমার Application এ JWT ব্যবহার করবা।

JWT Secret:

```txt
my-super-secret-key
```

এটা code এর মধ্যে লিখে রাখলে খারাপ practice।

# Bad Example

```java
@Service
public class JwtService {

    private String secret =
        "my-super-secret-key";
}
```

Problem:

```txt
Secret code এর ভিতরে hardcoded
```

---

তাই Config File এ রাখি।

```yaml
jwt:
  secret: my-super-secret-key
```

# @Value দিয়ে Load

```java
@Service
public class JwtService {

    @Value("${jwt.secret}")
    private String secret;
}
```

---

Spring Startup এ এটা করে:

```text
application.yml

jwt.secret
      │
      ▼
my-super-secret-key
      │
      ▼
secret variable এ inject
```

# Spring ভিতরে কী করে?

ধরো:

```java
@Service
public class JwtService {

    @Value("${jwt.secret}")
    private String secret;
}
```

Spring Startup এ:

```java
String value =
   readFromYaml(
      "jwt.secret"
   );
```

---

তারপর:

```java
jwtService.secret =
      value;
```

---

Equivalent Plain Java

```java
JwtService service =
      new JwtService();

service.secret =
      "my-super-secret-key";
```

# @Bean vs @Value

অনেক Beginner এখানে Confuse হয়।

## @Bean

```java
@Bean
public ModelMapper modelMapper() {
    return new ModelMapper();
}
```

কাজ:

```text
Object Create করে
Container এ Store করে
```

---

## @Value

```java
@Value("${jwt.secret}")
private String secret;
```

কাজ:

```text
Config Value Read করে
Variable এ বসায়
```

# Spring Core Heart View

এখন পর্যন্ত:

```text
@Component
@Service
@Repository
        │
        ▼
Bean Create


@Configuration
        │
        ▼
Bean Factory Class


@Bean
        │
        ▼
Custom Object Create


@Value
        │
        ▼
Config Value Inject


@RequiredArgsConstructor
        │
        ▼
Constructor Generate


Spring Container
        │
        ▼
সব Bean Store করে


Dependency Injection
        │
        ▼
এক Bean কে অন্য Bean এর মধ্যে দেয়
```

এখন তোমার জন্য সবচেয়ে গুরুত্বপূর্ণ পরের concept হলো **ApplicationContext (Spring Container) + Bean Lifecycle**। এটা বুঝলে "object কোথায় থাকে, কখন create হয়, destroy হয়, singleton কেন হয়" — সব পরিষ্কার হয়ে যাবে।
