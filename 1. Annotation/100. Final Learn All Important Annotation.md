# 1. Core Annotations

## @SpringBootApplication

**কি:** Spring Boot অ্যাপ শুরু করার main class।
**কোথায়:** Main class এর উপরে।
**কেন:** এটা ৩টা annotation এর কম্বিনেশন — `@Configuration` + `@ComponentScan` + `@EnableAutoConfiguration`। এটা ছাড়া Spring app ই চলবে না।

```java
@SpringBootApplication
public class LmsApplication {
    public static void main(String[] args) {
        SpringApplication.run(LmsApplication.class, args);
    }
}
```

**Laravel equivalent:** এটা Laravel এর `bootstrap/app.php` এর মতো — অ্যাপ শুরু করে।

---

## @Configuration

**কি:** এই class এ Spring Bean তৈরি হবে।
**কোথায়:** Config class এর উপরে।
**কেন:** যখন manually Bean রেজিস্টার করতে হয় (Security, Redis, Swagger etc)।

```java
@Configuration
public class RedisConfig {

    @Bean
    public RedisCacheManager cacheManager(RedisConnectionFactory factory) {
        // Redis কনফিগ করা হচ্ছে
        return RedisCacheManager.builder(factory).build();
    }
}
```

**Laravel equivalent:** `config/*.php` ফাইল বা `AppServiceProvider` এর `register()` method.

---

## @Bean

**কি:** এই method থেকে একটা object Spring container এ রেজিস্টার হবে।
**কোথায়:** @Configuration class এর ভিতরের method এ।
**কেন:** Third-party object (RestTemplate, PasswordEncoder etc) Spring এ inject করতে।

```java
@Configuration
public class AppConfig {

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @Bean
    public ModelMapper modelMapper() {
        return new ModelMapper();
    }
}
```

**Laravel equivalent:** `app()->singleton(PasswordEncoder::class, fn() => new BCryptPasswordEncoder())`

---

## @Value

**কি:** application.yml বা .properties থেকে ভ্যালু পড়ে।
**কোথায়:** Field বা constructor parameter এ।
**কেন:** Config value গুলো code এর বাইরে রাখতে (DB password, API key etc)।

```yaml
# application.yml
jwt:
  secret: my-secret-key
  expiration: 86400000

file:
  upload-dir: /uploads
  max-size: 26214400
```

```java
@Service
public class JwtService {

    @Value("${jwt.secret}")
    private String secret;                    // "my-secret-key" পাবে

    @Value("${jwt.expiration}")
    private long jwtExpiration;               // 86400000 পাবে

    @Value("${file.upload-dir:/default-path}")  // default value ও দেওয়া যায়
    private String uploadDir;
}
```

**Laravel equivalent:** `config('jwt.secret')` বা `env('JWT_SECRET')`

# 2. JPA / Entity Annotations

## @Entity

**কি:** এই Java class একটা database table কে represent করে।
**কোথায়:** Class এর উপরে।
**কেন:** JPA বুঝবে এই class এর জন্য table বানাতে হবে।

```java
@Entity
public class User {
    // এই class এর জন্য "user" table তৈরি হবে
}
```

**Laravel equivalent:** `class User extends Model`

---

## @Table

**কি:** Table এর নাম custom করতে।
**কোথায়:** Class এর উপরে, @Entity এর সাথে।
**কেন:** Class name আর table name আলাদা হলে (Java এ `Case` reserved word, table `cases` হবে)।

```java
@Entity
@Table(name = "users")                    // table name = "users"
public class User { }

@Entity
@Table(name = "cases")                    // Java এ "Case" reserved, তাই "cases"
public class CaseEntity { }

@Entity
@Table(name = "case_hearings",
       uniqueConstraints = @UniqueConstraint(columnNames = {"case_id", "hearing_date"}))
public class CaseHearing { }              // unique constraint ও দেওয়া যায়
```

**Laravel equivalent:** `protected $table = 'users';`

---

## @Id

**কি:** এটা primary key field।
**কোথায়:** Primary key field এর উপরে।
**কেন:** JPA কে বলে এটা PK।

```java
@Entity
public class User {
    @Id
    private Long id;              // এটা primary key
}
```

---

## @GeneratedValue

**কি:** Primary key auto-generate হবে।
**কোথায়:** @Id field এর পরে।
**কেন:** Manual ID দিতে হবে না, database auto increment করবে।

```java
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)   // auto increment
    private Long id;
}
```

**Laravel equivalent:** `$table->id()` বা `$table->bigIncrements('id')`

---

## @Column

**কি:** Column এর details কনফিগ করতে।
**কোথায়:** Field এর উপরে।
**কেন:** Nullable, unique, length, default value সেট করতে।

```java
@Entity
public class User {
    @Column(name = "email", nullable = false, unique = true, length = 150)
    private String email;                 // NOT NULL, UNIQUE, VARCHAR(150)

    @Column(name = "phone", length = 20)
    private String phone;                 // NULL OK, VARCHAR(20)

    @Column(name = "status", length = 20)
    private String status = "ACTIVE";     // default value = "ACTIVE"

    @Column(name = "amount", precision = 18, scale = 2)
    private BigDecimal amount;            // DECIMAL(18,2)

    @Column(name = "is_active")
    private Boolean active = true;        // BOOLEAN default true

    @Column(name = "remarks", columnDefinition = "TEXT")
    private String remarks;               // TEXT type
}
```

**Laravel equivalent:**
```php
$table->string('email', 150)->nullable(false)->unique();
$table->string('phone', 20)->nullable();
$table->decimal('amount', 18, 2);
$table->boolean('is_active')->default(true);
$table->text('remarks')->nullable();
```

---

## @Enumerated

**কি:** Java Enum কে database এ কিভাবে store করবে।
**কোথায়:** Enum field এর উপরে।
**কেন:** Enum কে integer না string হিসেবে save করবে তা বলতে। **সবসময় STRING ব্যবহার করো** — integer পড়তে কঠিন।

```java
public enum CaseStatus {
    DRAFT, FILED, ACTIVE, DECRED, CLOSED
}

@Entity
public class CaseEntity {
    @Enumerated(EnumType.STRING)           // DB তে "DRAFT", "FILED" ইত্যাদি string হিসেবে সেভ
    @Column(name = "status", length = 30)
    private CaseStatus status;

    @Enumerated(EnumType.ORDINAL)          // DB তে 0, 1, 2... integer হিসেবে সেভ (AVOID THIS!)
    private CaseStatus badExample;
}
```

**DB তে দেখাবে:**
```sql
-- EnumType.STRING → status = 'ACTIVE'    ← পড়তে সহজ
-- EnumType.ORDINAL → status = 2          ← 2 মানে কি? বুঝতে কোড দেখতে হবে
```

---

## @CreationTimestamp / @UpdateTimestamp

**কি:** created_at আর updated_at automatically সেট হবে।
**কোথায়:** Date field এর উপরে।
**কেন:** Manual date set করতে হবে না।

```java
@Entity
public class User {
    @CreationTimestamp
    @Column(name = "created_at", updatable = false)   // শুধু প্রথমবার সেট, পরে change হবে না
    private LocalDateTime createdAt;

    @UpdateTimestamp
    @Column(name = "updated_at")                       // প্রতি update এ বদলাবে
    private LocalDateTime updatedAt;
}
```

**Laravel equivalent:** `$table->timestamps()` — Laravel automatically করে, Spring এ manually করতে হয় এই annotations দিয়ে।

---

## @MappedSuperclass

**কি:** এটা parent class — নিজের table নেই, children এর table এ field গুলো add হবে।
**কোথায়:** Base/parent class এর উপরে।
**কেন:** Common fields (id, created_at, updated_at) সব entity তে না লিখে একবারে করতে।

```java
@MappedSuperclass
@Getter @Setter
public abstract class BaseEntity {

    @CreationTimestamp
    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @UpdateTimestamp
    @Column(name = "updated_at")
    private LocalDateTime updatedAt;

    @Column(name = "is_deleted")
    private Boolean deleted = false;
}

// এখন সব entity এ automatically created_at, updated_at, is_deleted যোগ হবে
@Entity
public class User extends BaseEntity {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    // created_at, updated_at, is_deleted automatically পাওয়া যাচ্ছে!
}

@Entity
public class CaseEntity extends BaseEntity {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String caseNo;
    // এখানেও created_at, updated_at, is_deleted আছে!
}
```

**Laravel equivalent:** Laravel automatically timestamps দেয়, কিন্তু custom base trait ও বানানো যায়।

---

## @Where / @SQLDelete (Soft Delete)

**কি:** Hard delete না করে soft delete (is_deleted = true) করবে।
**কোথায়:** Entity class এর উপরে।
**কেন:** Bank system এ data কখনো truly delete হবে না — regulation requirement।

```java
@Entity
@Table(name = "cases")
@Where(clause = "is_deleted = false")
@SQLDelete(sql = "UPDATE cases SET is_deleted = true, deleted_at = NOW() WHERE id = ?")
public class CaseEntity extends BaseEntity {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String caseNo;

    @Column(name = "deleted_at")
    private LocalDateTime deletedAt;
}

// ব্যবহার:
caseRepository.delete(caseEntity);
// SQL: UPDATE cases SET is_deleted = true, deleted_at = NOW() WHERE id = 5
// DELETE FROM cases WHERE id = 5 ← এটা আর হবে না!

caseRepository.findAll();
// SQL: SELECT * FROM cases WHERE is_deleted = false
// মুছে যাওয়া data আর দেখাবে না!
```

**Laravel equivalent:** `use SoftDeletes;` trait + `$table->softDeletes()`

# 3. JPA Relation Annotations

## @OneToMany

**কি:** একটা parent এর অনেকগুলো child আছে। (1 → N)
**কোথায়:** Collection field এর উপরে parent entity তে।
**কেন:** Branch এ অনেক User, Case এ অনেক Hearing — এই সম্পর্ক বোঝাতে।

```java
// Branch (1) → Users (N)
@Entity
public class Branch {
    @Id
    private Long id;
    private String name;

    @OneToMany(mappedBy = "branch", cascade = CascadeType.ALL, fetch = FetchType.LAZY)
    private List<User> users = new ArrayList<>();
    //      ↑ mappedBy = "branch" মানে User class এ যে field এর নাম "branch"
}

@Entity
public class User {
    @Id
    private Long id;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "branch_id")     // DB তে branch_id column তৈরি হবে
    private Branch branch;
}
```

**DB:**
```sql
-- branches table: id, name
-- users table: id, name, branch_id (FK → branches.id)
```

**Laravel equivalent:**
```php
// Branch.php
public function users() { return $this->hasMany(User::class); }
// User.php
public function branch() { return $this->belongsTo(Branch::class); }
```

---

## @ManyToOne

**কি:** অনেক child একটা parent এর। (N → 1)
**কোথায়:** Single object field এর উপরে child entity তে।
**কেন:** অনেক User একটা Branch এ, অনেক Case একটা Customer এর।

```java
@Entity
public class CaseEntity {
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "customer_id")     // cases table এ customer_id column
    private Customer customer;
}
```

---

## @ManyToMany

**কি:** অনেক A এর অনেক B, অনেক B এর অনেক A। (N ↔ N)
**কোথায়:** Set/List field এর উপরে দুই entity তেই।
**কেন:** User অনেক Role এ, Role অনেক User এ — pivot table লাগে।

```java
@Entity
public class User {
    @ManyToMany(fetch = FetchType.LAZY)
    @JoinTable(
        name = "user_roles",                                    // pivot table name
        joinColumns = @JoinColumn(name = "user_id"),            // এই entity এর FK
        inverseJoinColumns = @JoinColumn(name = "role_id")      // ওই entity এর FK
    )
    private Set<Role> roles = new HashSet<>();
}

@Entity
public class Role {
    @ManyToMany(mappedBy = "roles", fetch = FetchType.LAZY)
    private Set<User> users = new HashSet<>();
}
```

**DB:**
```sql
-- users table: id, name
-- roles table: id, name
-- user_roles table: user_id (FK), role_id (FK)    ← pivot table
```

**Laravel equivalent:** `belongsToMany(Role::class)` + `role_user` pivot table

---

## @OneToOne

**কি:** একটার একটাই। (1 ↔ 1)
**কোথায়:** Single object field এর উপরে।
**কেন:** Case এর একটাই Financial Summary, একটা User এর একটাই Profile।

```java
@Entity
public class CaseEntity {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @OneToOne(mappedBy = "caseEntity", cascade = CascadeType.ALL, fetch = FetchType.LAZY)
    private CaseFinancialSummary financialSummary;
}

@Entity
public class CaseFinancialSummary {
    @Id
    private Long id;                             // এটাই case_id (shared PK)

    @OneToOne(fetch = FetchType.LAZY)
    @MapsId                                      // caseEntity.id = এই class এর id (shared)
    @JoinColumn(name = "case_id")
    private CaseEntity caseEntity;

    private BigDecimal totalClaimAmount;
    private BigDecimal netPosition;
}
```

**DB:**
```sql
-- cases table: id, case_no
-- case_financial_summary table: case_id (PK + FK → cases.id), total_claim_amount, net_position
```

---

## @JoinColumn

**কি:** Foreign key column এর নাম বলে দেয়।
**কোথায়:** @ManyToOne, @OneToOne field এর উপরে।
**কেন:** DB তে FK column এর নাম কি হবে তা specify করতে।

```java
@ManyToOne
@JoinColumn(name = "branch_id")           // DB তে "branch_id" column তৈরি হবে
private Branch branch;

@ManyToOne
@JoinColumn(name = "court_id")            // DB তে "court_id" column তৈরি হবে
private Court court;
```

---

## @JoinTable

**কি:** ManyToMany এ pivot table এর নাম বলে দেয়।
**কোথায়:** @ManyToMany field এর উপরে।
**কেন:** Pivot table এর নাম আর column গুলো define করতে।

```java
@ManyToMany
@JoinTable(
    name = "user_roles",                                // pivot table
    joinColumns = @JoinColumn(name = "user_id"),        // current entity এর FK
    inverseJoinColumns = @JoinColumn(name = "role_id")  // opposite entity এর FK
)
private Set<Role> roles;
```

---

## @MapsId

**কি:** OneToOne এ primary key ই foreign key (shared PK)।
**কোথায়:** @OneToOne field এর উপরে।
**কেন:** আলাদা FK column না করে, PK কেই FK হিসেবে ব্যবহার করতে।

```java
@Entity
public class CaseFinancialSummary {
    @Id
    private Long id;                       // এটা = case_id

    @OneToOne
    @MapsId                               // id = caseEntity.id (same value)
    private CaseEntity caseEntity;
}
// case_financial_summary table: id (=case_id) → আলাদা case_id column লাগবে না
```

## FetchType.LAZY vs EAGER

**কি:** Related data কখন load হবে।
**কেন:** Performance এর জন্য। **সবসময় LAZY ব্যবহার করো।**

```java
// LAZY = যখন দরকার তখন load হবে (GOOD ✓)
@OneToMany(mappedBy = "branch", fetch = FetchType.LAZY)
private List<User> users;

// EAGER = parent load হলেই child ও load হবে (BAD ✗ → N+1 problem)
@OneToMany(mappedBy = "branch", fetch = FetchType.EAGER)
private List<User> users;
```

**ব্যাখ্যা:**
```text
EAGER: branch = repository.findById(1);
       → SQL: SELECT * FROM branches WHERE id=1;
       → SQL: SELECT * FROM users WHERE branch_id=1;     ← অটোমেটিক!
       Problem: 100 branch load করলে 101 query চলবে!

LAZY:  branch = repository.findById(1);
       → SQL: SELECT * FROM branches WHERE id=1;         ← শুধু branch
       branch.getUsers();                                  ← এখন users load হবে
       → SQL: SELECT * FROM users WHERE branch_id=1;     ← যখন call করবে
```

## CascadeType

**কি:** Parent এর উপর action হলে child এও কি হবে।
**কেন:** Parent delete করলে children ও delete হবে কিনা তা বলতে।

```java
@OneToMany(mappedBy = "case", cascade = CascadeType.ALL)        // সব action cascade
private List<CaseHearing> hearings;

// CascadeType.ALL       = সব (save, delete, update, refresh, detach)
// CascadeType.PERSIST   = parent save করলে children ও save
// CascadeType.MERGE     = parent update করলে children ও update
// CascadeType.REMOVE    = parent delete করলে children ও delete
// CascadeType.DETACH    = parent detach করলে children ও detach

// orphanRemoval = true → list থেকে remove করলে DB থেকে delete
@OneToMany(mappedBy = "case", cascade = CascadeType.ALL, orphanRemoval = true)
private List<CaseHearing> hearings;
```

## @RestControllerAdvice

**কি:** Global exception handler — সব controller এর error এখানে ধরা হবে।
**কোথায়:** Exception handler class এর উপরে।
**কেন:** প্রতিটা controller এ try-catch না লিখে এক জায়গায় error handle করতে।

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ApiResponse<Void>> handleNotFound(ResourceNotFoundException ex) {
        return ResponseEntity.status(404)
            .body(ApiResponse.error(ex.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiResponse<Map<String, String>>> handleValidation(
            MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors()
            .forEach(e -> errors.put(e.getField(), e.getDefaultMessage()));
        return ResponseEntity.badRequest()
            .body(new ApiResponse<>(false, "Validation failed", errors));
    }
}
```

## @Transactional

**কি:** এই method এর সব DB operation একটা transaction এ চলবে।
**কোথায়:** Service method এর উপরে।
**কেন:** কোনো step fail করলে সব rollback হবে। Bank system এ MUST HAVE।

```java
@Service
public class PaymentService {

    // সব operation একসাথে success বা একসাথে fail
    @Transactional
    public void transferMoney(Long fromId, Long toId, BigDecimal amount) {
        Account from = accountRepository.findById(fromId).orElseThrow();
        Account to = accountRepository.findById(toId).orElseThrow();

        from.setBalance(from.getBalance().subtract(amount));
        to.setBalance(to.getBalance().add(amount));

        accountRepository.save(from);
        // যদি এখানে error হয় → from এর balance ও revert হবে!
        accountRepository.save(to);
    }

    // শুধু read করলে readOnly = true দাও (performance better)
    @Transactional(readOnly = true)
    public List<CaseResponse> getCases() {
        return caseRepository.findAll();
    }

    // কোন exception এ rollback হবে
    @Transactional(rollbackFor = Exception.class)
    public void createCase() { }
}
```

**Laravel equivalent:** `DB::transaction(function() { ... })`

# 8. Event / Async Annotations

## @EventListener

**কি:** এই method একটা event listen করবে। event fire হলে auto-call হবে।
**কোথায়:** Method এর উপরে @Component class এ।

```java
@Component

public class CaseEventListener {

// Event fire করা:
@Service
@RequiredArgsConstructor
public class CaseService {
    private final ApplicationEventPublisher eventPublisher;

    public void createCase() {
        // ... case create logic ...
        eventPublisher.publishEvent(new CaseCreatedEvent(caseId, caseNo));
        // এখান থেকে সব @EventListener method auto-call হবে!
    }
}
```

**Laravel equivalent:**
```php
// event(new CaseCreated($case));
// class CaseCreatedListener { public function handle(CaseCreated $event) { } }
```

## @Async

**কি:** এই method background thread এ চলবে, main thread block হবে না।
**কোথায়:** Method এর উপরে (বা @EventListener এর সাথে)।
**কেন:** Email/SMS পাঠাতে সময় লাগে — user কে wait করাতে না।

# 9. Caching Annotations

## @Cacheable

**কি:** Method এর result cache করে। পরের বার same parameter এ call করলে DB query না করে cache থেকে দেবে।
**কোথায়:** Service method এর উপরে।
**কেন:** Dashboard data, expensive query — বারবার query না করে cache থেকে দিতে।

```java
@Service
public class DashboardService {

    @Cacheable(value = "dashboard", key = "#branchId ?: 'all'")   // 30 min cache
    public DashboardResponse getDashboard(Long branchId) {
        // প্রথমবার: DB query করবে, result cache করবে
        // দ্বিতীয়বার (30 min এর মধ্যে): DB query না করে cache থেকে return
        return calculateDashboard(branchId);
    }
}

// @Cacheable(value = "users", key = "#id")                    → user:{id}
// @Cacheable(value = "cases", key = "#status + '_' + #page")  → cases:{status}_{page}
```

**Laravel equivalent:** `Cache::remember('dashboard', 1800, fn() => calculate())`

---

## @CacheEvict

**কি:** Cache clear করে।
**কোথায়:** Service method এর উপরে।
**কেন:** Data change হলে পুরানো cache invalid করতে।

```java
@Service
public class CaseService {

    }

    @CacheEvict(value = "cases", key = "#id")               // specific case cache clear
    public CaseResponse updateCase(Long id) { }
}
```

**Laravel equivalent:** `Cache::forget('dashboard')` বা `Cache::flush()`

# 10. Scheduling Annotations

## @Scheduled

**কি:** এই method automatically নির্দিষ্ট সময়ে চলবে।
**কোথায়:** @Component class এর method এর উপরে।
**কেন:** Cron job — daily report, reminder, auto-expire etc।

```java
@Component
@RequiredArgsConstructor
public class LmsScheduledTasks {

    // fixedRate — প্রতি 5 minutes (৩০০০০ ms)
    @Scheduled(fixedRate = 300000)
    public void syncCbsData() {
        // প্রতি 5 minutes CBS থেকে data sync
    }

    // cron expression — নির্দিষ্ট সময়ে
    // format: second minute hour day month weekday
    @Scheduled(cron = "0 7 * * *")              // প্রতিদিন সকাল 7:00
    public void sendMorningReminders() { }

    @Scheduled(cron = "0 0 * * *")              // প্রতি ঘন্টায়
    public void hourlyCheck() { }

    @Scheduled(cron = "0 2 1 * *")              // প্রতি মাসের ১ তারিখে রাত 2:00
    public void monthlyReport() { }

    @Scheduled(cron = "0 0 9 * * MON-FRI")      // সোম-শুক্র সকাল 9:00
    public void weekdayMorningTask() { }
}
```

**Cron format:**
```text
┌─────────── second (0-59)
│ ┌─────────── minute (0-59)
│ │ ┌─────────── hour (0-23)
│ │ │ ┌─────────── day of month (1-31)
│ │ │ │ ┌─────────── month (1-12)
│ │ │ │ │ ┌─────────── day of week (0-6, 0=Sunday)
│ │ │ │ │ │
* * * * * *
```

**Laravel equivalent:** `app/Console/Kernel.php` এর `schedule()` method

```php
// Laravel
$schedule->command('reminders:send')->dailyAt('07:00');
```

# 11. Audit / Lifecycle Annotations

## @Audited

**কি:** এই entity এর সব change track হবে (Spring Data Envers)।
**কোথায়:** Entity class বা specific field এর উপরে।
**কেন:** Audit trail — কে কখন কি change করেছে তার ইতিহাস।

```java
@Entity
@Audited                                // সব field এর change track হবে
public class CaseEntity {
    @Id
    private Long id;
    private String status;              // status change হলে audit entry তৈরি হবে
    private BigDecimal claimAmount;     // amount change হলে audit entry তৈরি হবে

    @NotAudited                         // এই field track হবে না
    private LocalDateTime updatedAt;
}
```

---

## @PostPersist / @PostUpdate

**কি:** Entity save/update হওয়ার পরে automatically কোড চলায়।
**কোথায়:** Entity class এর method এর উপরে।
**কেন:** Auto audit log, notification trigger।

```java
@Entity
public class CaseEntity {

    @PostPersist                          // INSERT এর পরে auto-call
    public void afterCreate() {
        System.out.println("New case created: " + this.caseNo);
    }

    @PostUpdate                           // UPDATE এর পরে auto-call
    public void afterUpdate() {
        System.out.println("Case updated: " + this.id);
    }
}
```

# 13. WebSocket Annotations

## @EnableWebSocketMessageBroker

**কি:** WebSocket + STOMP protocol enable করে।
**কোথায়:** Config class এর উপরে।

```java
@Configuration
```

## @Controller (WebSocket)

**কি:** WebSocket message handle করতে @Controller ব্যবহার হয় (@RestController না)।
**কোথায়:** WebSocket message handler class এর উপরে।

```java
@Controller
public class WebSocketController {

    @MessageMapping("/notification.read")     // client এই path তে message পাঠালে
    public void markAsRead(@Payload Map<String, Long> payload) {
        // handle message from client
    }
}
```

---

## @MessageMapping

**কি:** WebSocket message এর destination path।
**কোথায়:** Method এর উপরে।
**কেন:** Client যে path এ message পাঠাবে তা define করতে।

```java
@Controller
public class WebSocketController {

    @MessageMapping("/case.subscribe")         // client sends to /app/case.subscribe
    public void subscribeToCase(@Payload Map payload) { }
}
```

---

## @Payload

**কি:** WebSocket message এর body/data কে Java object তে convert করে।
**কোথায়:** Method parameter এ।

```java
@MessageMapping("/notification.read")
public void markAsRead(@Payload Map<String, Long> payload) {
    Long notificationId = payload.get("id");
}
```

## @SendTo / @SendToUser

**কি:** Method এর response কোন WebSocket channel এ যাবে।
**কোথায়:** Method এর উপরে।

```java
@Controller
public class NotificationController {

    @MessageMapping("/chat.send")
    @SendTo("/topic/messages")                       // broadcast to all
    public ChatMessage send(ChatMessage message) {
        return message;
    }

    @MessageMapping("/private.message")
    @SendToUser("/queue/messages")                   // only to specific user
    public ChatMessage privateMessage(ChatMessage message) {
        return message;
    }
}
```

# 14. Lombok Annotations

> Lombok = boilerplate code অটোমেটিক তৈরি করে (getter, setter, constructor etc)

## @Getter / @Setter

**কি:** getter আর setter method automatically বানায়।
**কোথায়:** Class বা field এর উপরে।
**কেন:** Manual getter/setter না লিখতে।

```java
// ছাড়া Lombok:
public class User {
    private String name;
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    // প্রতিটা field এ ৬ লাইন!
}

// Lombok দিয়ে:
@Getter @Setter
public class User {
    private String name;           // getter + setter auto-generated
    private String email;          // getter + setter auto-generated
    private String phone;          // getter + setter auto-generated
}
```

---

## @RequiredArgsConstructor

**কি:** `final` field গুলো দিয়ে constructor automatically বানায়।
**কোথায়:** Class এর উপরে।
**কেন:** Dependency injection এর জন্য constructor না লিখতে। **সবচেয়ে বেশি ব্যবহৃত।**

```java
// ছাড়া Lombok:
@Service
public class CaseService {
    private final CaseRepository caseRepository;
    private final CustomerRepository customerRepository;
    private final CaseMapper caseMapper;

    public CaseService(CaseRepository caseRepository,
                       CustomerRepository customerRepository,
                       CaseMapper caseMapper) {
        this.caseRepository = caseRepository;
        this.customerRepository = customerRepository;
        this.caseMapper = caseMapper;
    }
}

// Lombok দিয়ে:
@Service
@RequiredArgsConstructor               // constructor auto-generated!
public class CaseService {
    private final CaseRepository caseRepository;
    private final CustomerRepository customerRepository;
    private final CaseMapper caseMapper;
    // এটাই! constructor লেখার দরকার নেই
}
```

---

## @Slf4j

**কি:** Logger object automatically তৈরি করে।
**কোথায়:** Class এর উপরে।
**কেন:** Manual logger না বানাতে।

```java
// ছাড়া Lombok:
public class CaseService {
    private static final Logger log = LoggerFactory.getLogger(CaseService.class);
    public void createCase() { log.info("Creating case"); }
}

// Lombok দিয়ে:
@Slf4j
public class CaseService {
    public void createCase() { log.info("Creating case"); }
}
```

## @Builder

**কি:** Builder pattern automatically বানায়।
**কোথায়:** Class বা record এর উপরে।
**কেন:** Object তৈরি সহজ আর readable হয়।

```java
@Getter @Setter @Builder
public class NotificationPayload {
    private Long id;
    private String title;
    private String message;
    private String type;
    private boolean isRead;
    private LocalDateTime createdAt;
}

// ব্যবহার:
NotificationPayload payload = NotificationPayload.builder()

    .title("Case Assigned")
    .message("Case LMS/2024/001 assigned to you")
    .type("CASE_ASSIGNMENT")
    .isRead(false)
    .createdAt(LocalDateTime.now())
    .build();
```

## @CircuitBreaker / @Retry (Resilience4j)

**কি:** External API call fail হলে fallback/retry logic।
**কোথায়:** Service method এর উপরে।
**কেন:** CBS API down হলে অ্যাপ crash হবে না।

```java
@Service
public class CbsService {

    @CircuitBreaker(name = "cbs", fallbackMethod = "fallback")
    @Retry(name = "cbs")
    public CbsResponse getCustomer(String accountNo) {
        return restTemplate.getForObject("/customers/" + accountNo, CbsResponse.class);
    }

    // CBS down হলে এই method call হবে
    public CbsResponse fallback(String accountNo, Exception e) {
        throw new ServiceUnavailableException("CBS temporarily unavailable");
    }
}
```

# সংক্ষিপ্ত তালিকা — সব Annotation এক জায়গায়

## সবচেয়ে বেশি ব্যবহৃত (Daily Use — এগুলো অবশ্যই মনে রাখো)

```text
@Service              → Business logic class
@Repository           → Database query interface
@RestController       → API endpoint class
@Entity               → Database table mapping
@Transactional        → DB transaction wrapper
@RequiredArgsConstructor → Constructor injection (Lombok)
@Getter @Setter       → Auto getter/setter (Lombok)
@Slf4j                → Auto logger (Lombok)
@Valid                → Input validation trigger
@Autowired            → Dependency injection (avoid, use constructor)
```

## JPA Relation (Database)

```text
@OneToMany            → 1 এর অনেক (Branch → Users)
@ManyToOne            → অনেকের 1 (Users → Branch)
@ManyToMany           → অনেক ↔ অনেক (User ↔ Role)
@OneToOne             → 1 ↔ 1 (Case ↔ FinancialSummary)
@JoinColumn           → FK column name
@JoinTable            → Pivot table config
@MapsId               → Shared PK (OneToOne)
```

## Web / API

```text
@GetMapping           → READ endpoint

@DeleteMapping        → DELETE endpoint
@PathVariable         → URL এর {id} part
@RequestParam         → Query parameter (?page=1)
@RequestBody          → JSON body → Java object
@RestControllerAdvice → Global error handler
@ExceptionHandler     → Specific error handler
```

## Validation

```text
@NotBlank             → null/empty/blank না
@NotNull              → null না
@Email                → valid email
@DecimalMin           → minimum number

```

## Security
