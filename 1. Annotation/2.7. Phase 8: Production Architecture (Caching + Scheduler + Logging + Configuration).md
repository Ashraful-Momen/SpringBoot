# Part 1: Caching Architecture

## Why Cache?

Problem:

```txt
User Request

     │

     ▼

Database

     │

     ▼

Response
```

Every Request Database Hit করে।

Example:

```txt
1000 Users

1000 DB Queries

Slow Response
```

---

## Solution

```txt
User Request

      │

      ▼

Cache Check

      │

Found ?

YES          NO

 │            │

 ▼            ▼

Cache      Database

 │            │

 ▼            ▼

Response   Save Cache
```

---

# Cache Architecture

```txt
Client

   │

   ▼

   ▼

Service

   │

   ▼

Cache

 │      │

Hit    Miss

 │      │

 ▼      ▼

        │

        ▼

     Save Cache
```

# @Cacheable

Used For:

```txt
Read Operation
```

---

## StudentService.java

```java
@Service
public class StudentService {

    @Autowired
    private StudentRepository repository;

    @Cacheable(
            value = "students",
            key = "#id"
    )
    public Student getStudent(
            Long id
    ){

        System.out.println(
            "Fetching From DB"
        );

        return repository
                .findById(id)
                .orElseThrow();
    }
}
```

---

# @CacheEvict

Used For:

```txt

When Data Changes
```

---

```java

    value = "students",
    key = "#id"
)

        Long id
){

    repository.deleteById(id);
}
```

# @Scheduled

Run Automatically.

---

## ReportScheduler.java

```java
@Component
@Slf4j
public class ReportScheduler {

    @Scheduled(
            fixedRate = 60000
    )
    public void generateReport(){

        log.info(
            "Generating Report..."
        );
    }
}
```

---

# fixedRate

```txt
Every 60 Seconds
```

---

# fixedDelay

```java
@Scheduled(
        fixedDelay = 5000
)
public void processQueue(){

}
```

Flow:

```txt

      │

      ▼

      │

      ▼

Run Again
```
